\chapter{QAOA}
In the previous chapters we focused on obtaining a Hamiltonian whose ground state represented the solution to our problem. Implicitly or explicitly we referred to quantum annealers as machines for finding the configuration that minimizes the energy.

In this chapter we will show how to move from a problem formulated for quantum annealers to a problem formulated for quantum gate based computers through the \emph{Quantum Approximation Optimization Algorithm} (QAOA), proposed for the first time by Farhi et al. in~\cite{farhi2014quantum}.

We begin by describing the QAOA algorithm, highlighting theoretical advantages and disadvantages with respect to QAC\footnote{Quantum Adiabatic Computing (Section~\ref{}).}. We will then show how it is possible to translate a problem formulated in QUBO form (Section~\ref{}), obtained at the end of the QA-Prolog pipeline, into a problem suitable for IBM quantum gate based computers.

Finally, we will test the proposed translation and present some experimental results, discussing them.
\section{QAOA}
The QAOA algorithm is inspired by QAC; both attempt to find the minimum eigenvalue of a matrix that represents the energy of a physical system~\cite{fakhimi2023quantum}. This matrix describes an observable, consequently it is a Hermitian operator and in particular it is the Hamiltonian $\mathbf{H}$ of the system. The smallest eigenvalue of $\mathbf{H}$ is the minimum energy of the system that can be measured; for this reason it is called the ground state energy.

As described in Section~\ref{} QAC is executed on a quantum annealer that starts from a Hamiltonian whose ground state is easy to know and evolves, adiabatically (to remain in the ground state), towards the Hamiltonian with unknown ground state~\cite{fakhimi2023quantum}. The evolution time from one Hamiltonian to another can be exponential\cite{farhi2014quantum}; moreover, generally the success probability of QAC does not increase as the annealing time increases.

QAOA targets quantum gate based machines and, once its parameters are optimized, the quality of the result increases as the number of iterations, called \emph{levels}, increases.

\subsection{Circuit}
\label{sec:qaoa_circ}
The QAOA algorithm begins with a state $\Ket{s}$ with maximum superposition. It is possible to obtain this state starting from the state with $n$ \emph{qubits} set to zero ($\Ket{0}$) and applying the Hadamard gate ($H$) to each of them~\cite{fakhimi2023quantum}; we denote this operator with $H^{\otimes n}$. 

The circuit then consists of $p$ identical levels. Each level is composed of two operators: 
\begin{itemize}
	\item \emph{phase separation operator}: $U_P(\gamma) = e ^{-i\gamma\mathbf{H}}$ where $\mathbf{H}$ is the Hamiltonian of the system, and $\gamma \in [0, 2\pi]$ is a parameter;
	\item \emph{mixing operator}:  $U_M(\beta) = e ^{-i\beta B}$ where $B$ is the operator that applies the Pauli Gates $X$ to every \emph{qubit}, and $\beta \in [0, \pi]$ is a parameter.
\end{itemize}
The operator $U_P(\gamma)$ simulates $\mathbf{H}$, while $U_M(\beta)$, which does not commute with $U_P(\gamma)$\footnote{$U_P(\gamma) \times U_M(\beta) \neq U_M(\beta) \times U_P(\gamma)$.}, allows modifying the probability distribution of the output~\cite{fakhimi2023quantum}. 

Using Pauli Gates X is not the only way to construct the mixing operator; in \cite{wang2020xy, hadfield2019quantum} other mixing operators are explored.

\begin{figure}[h]
	\centering
	\includegraphics[width=.8\linewidth]{qaoa/qaoa}
	\caption{QAOA scheme~\cite{blekos2024review}}
	\label{fig:qaoa}
\end{figure}

Figure~\ref{fig:qaoa} shows the general scheme of the QAOA algorithm considering $p$ levels.

Given $\vec{\gamma} = (\gamma_1, \gamma_2,\dots,\gamma_p)$ and $\vec{\beta} = (\beta_1, \beta_2,\dots,\beta_p)$, we denote with $\Ket{\psi(\vec{\gamma},\vec{\beta})}$ the \emph{ansatz} (an estimate of the solution) of QAOA. The ansatz is defined as:
\begin{equation}
\Ket{\psi(\vec{\gamma},\vec{\beta})} = U_M(\beta_p)U_P(\gamma_p)\cdots U_M(\beta_1)U_P(\gamma_1)\Ket{s}.
\label{eq:qaoa_state}
\end{equation}
The circuit ends with the measurement; the expected value of this measurement is:
\begin{equation}
F(\vec{\gamma},\vec{\beta}) =\Braket{\psi(\vec{\gamma},\vec{\beta}) | \mathbf{H} | \psi(\vec{\gamma},\vec{\beta})}.
\label{eq:qaoa_value}
\end{equation}

$F(\vec{\gamma},\vec{\beta})$ represents the expected value of $\mathbf{H}$ in the state $\Ket{\psi(\vec{\gamma},\vec{\beta})}$~\cite{fakhimi2023quantum}. Once implemented this value is obtained by executing the quantum circuit many times, we will see this later. In addition to the value of the system energy, the measurement of the \emph{qubits} also provides an assignment of the variables that represents a possible solution to the problem.

The QAOA’s performance depends on the number of levels; given optimal parameters of QAOA, the algorithm asymptotically converges to optimal solutions as the number of levels grows\cite{farhi2014quantum}.

\subsection{Parameter optimization}
\label{sec:param_opt}
QAOA requires tuning the parameters to minimize the expected objective value for a given number of levels. Formally the best parameter ($\vec{\gamma^*}$ and $\vec{\beta^*}$) are defined as:
\[
(\vec{\gamma^*},\vec{\beta^*}) = \argmin_{\vec{\gamma},\vec{\beta}} F(\vec{\gamma},\vec{\beta}) 
\]
This parameter tuning problem is nonconvex and NP-hard~\cite{fakhimi2023quantum}. Even with one level, the gradient of $F(\vec{\gamma},\vec{\beta})$  vanishes rapidly; this phenomenon is known as the barren plateau~\cite{arrasmith2021effect, mcclean2018barren}. The barren plateau expands exponentially as the number of qubits
grows, making the tuning process even more challenging~\cite{mcclean2018barren}.

The parameter optimization process is executed on a classical CPU rather than on a quantum computer. This makes QAOA a hybrid algorithm~\cite{fakhimi2023quantum}.

\paragraph{Conclusion:} Despite the difficulty in optimizing the parameters, QAOA algorithm presents numerous interesting aspects and research in this field is very active. In our work it gives us the possibility to change the reference architecture, and attempt to solve a QUBO-formulated problem \textemdash{}obtained with the QA-Prolog pipeline\textemdash{} using quantum gate based computers. In the next sections we will see how to practically implement the QAOA algorithm.

\section{From QAC to QAOA}
In the previous section we focused on the theoretical aspects of the QAOA algorithm. In this section instead we show how to use IBM's Qiskit Framework to solve a minimization problem such as those obtained at the end of the QA-Prolog pipeline.

\subsection{Objective}
At the end of the QA-Prolog pipeline we obtain a problem in QUBO form: in this representation the variables $x_i$ to be optimized belong to the set $\set{0, 1}$. When we move to the quantum gate based architecture, the measured value of the \emph{qubits} belongs to the set $\set{-1, 1}$. We must therefore move from a representation in QUBO form to a problem in Ising form. Qiskit also requires that the matrix $\mathbf{H}$ is expressed as Pauli operators.

With these steps we have made the output of QA-Prolog compatible with the input of the QAOA algorithm. To complete the implementation we must then optimize the parameters defined in Equations~\ref{eq:qaoa_state}~and~\ref{eq:qaoa_value} in order to minimize the expected value of $\mathbf{H}$. Finally, after finding the optimal parameters we will sample the result of the quantum circuit one last time to obtain the result of our problem.

To summarize, to implement the QAOA algorithm we must:
\begin{itemize}
	\item move from QUBO form to Ising form;
	\item express the observable $\mathbf{H}$ using Pauli operators;
	\item optimize the parameters of Equation~\ref{};
	\item sample the optimized quantum circuit.
\end{itemize}

The most interesting aspect of this procedure is the removal of the constant term; this value is used to align the ground state of the problem in QUBO form and Ising form (Equation~\ref{}) to the same energy. Since our objective is to find the ground state, regardless of its value, we can ignore this offset and consider only the coefficients of the variables.

\subsection{Hamiltonian as Pauli operator}
Consider the upper triangular Hamiltonian $\mathbf{H}$ that describes our problem in Ising form. Each element belonging to the diagonal $h_{i,i}$ is translated into a gate that measures the $Z$ component of the \emph{qubit} $i$ and applies the identity transformation to all the other \emph{qubits}\footnote{The operator that measures the $Z$ component is $\sigma_z$ (Equation~\ref{eqn:pauli_op}); this operator is both Hermitian and Unitary, therefore it describes both an observable and a valid quantum gate. The same holds for the identity operator.}. The elements $h_{i,j}$\footnote{With $j>i$ because $\mathbf{H}$ is upper triangular.} are transformed into gates that measure the spin along the $Z$ axis of the \emph{qubits} $i$ and $j$, and apply the identity operator to the other \emph{qubits}.

Following \cite{ibm_qaoa_tutorial}, we use the Python library \mintinline{text}|scipy| to minimize $F(\vec{\gamma}, \vec{\beta})$.

\section{Experiments}
We apply the QAOA algorithm to solve a problem in QUBO form obtained with the QA-Prolog pipeline.
\subsection{Problem}
Consider the satisfiability problem $y = x_1 \land\neg(x_2 \lor x_3)$ (Equation~\ref{eqn:simple_logic}). This problem is well suited for the application of QAOA because we can read the solution directly from the value of the \emph{qubits}\footnote{We will discuss the possibility of using QA-Prolog to make the output of QAOA readable in \ref{}.}. 

As shown in Example~\ref{ex:sat_prolog}, starting from Prolog code to solve the satisfiability problem introduces a significant amount of overhead. To minimize the number of \emph{qubits} required to represent the problem we follow only part of the pipeline, starting from the Verilog code of Example~\ref{ex:sat_verilog}.

The QUBO problem that we obtain is represented by an $11\times11$ matrix, this means that we need eleven \emph{qubits} to encode our problem.

\subsection{Execution}
To solve the satisfiability problem we construct a QAOA with $10$ levels. By increasing the number of levels, the algorithm converges increasingly well to the optimal solution. 

However, increasing the number of levels also leads to longer execution time and, if executed on quantum hardware (and not through simulators), the deeper the circuit, the more one encounters the phenomenon of decoherence~\cite{bibid}. Moreover, the number of parameters to optimize is equal to twice the number of levels, therefore increasing the number of levels also makes the parameter optimization more complex\footnote{As sed in Section~\ref{sec:param_opt}}.

The obtained result is the expected one; in fact we obtained the string $1\ 0\ 0$ as the most probable sample. These values must be associated respectively with the variables $x_1$, $x_3$ and $x_2$, and we can easily verify that this assignment satisfies Equation~\ref{eqn:simple_logic}.

\subsection{Results}
Both the sampling process and the parameter optimization process introduce stochastic components into the execution of QAOA. We therefore want to verify, by executing the parameter optimization and the final sampling multiple times, if we always obtain the expected result\footnote{The code that automates the execution of the experiments is available at \url{https://github.com/DavideCamino/Thesis/blob/main/Code/3_QAUA/circ_sat/circ_sat_qaoa_multiple_run.ipynb}.}.

Since the execution time of each experiment needs to be taken into consideration, the number of experiments performed is limited and does not allow reliable statistical analysis. However, it is sufficient to show how the QAOA algorithm is not always able to find the correct answer to the minimization problem.

Figure~\ref{fig:qaoa_r_3} shows the results (of the relevant \emph{qubits}) of 12 repetitions of the experiment. In the image, all the cases in which the correct assignment of the variables $x_1$, $x_2$ and $x_3$ was identified are highlighted with a green. The cases, in which the most probable result is incorrect, are highlighted in red.

The number of experiments is very low and advanced statistical analyses would not be reliable. Nevertheless, we can make some qualitative observations. Typically, in the cases in which the correct solution is found, the probability of the latter is noticeably higher than that of the other results. When instead the solution is incorrect, the different results are distributed in a rather uniform way in over half of the cases.

\section{Conclusion}
In this chapter we presented a new algorithm for solving a minimization problem in QUBO form. From a theoretical point of view, we saw that QAOA guarantees a monotonic improvement in performance as the number of levels increases (provided that optimal parameters have been found), while QAC does not guarantee this monotonicity when increasing the annealing time. We then presented the critical aspects of QAOA, highlighting how the problem of finding optimal parameters is, in general, NP-Hard.

The discussion then moved to the practical implementation, and we saw how to use the Qiskit Framework to solve on IBM quantum gate based machines a problem originally in QUBO form. We presented Python scripts to automate the translation, we saw how to retrieve the results and how to interpret them. Finally, we discussed the quality of the solutions provided by QAOA by repeating the parameter tuning and the sampling of the optimized circuit, discussing the obtained results.

In the previous chapter we showed how to add a rewriting step upstream of the QA-Prolog pipeline. This chapter can also be considered as a further rewriting step of a problem into different but equivalent formats; this time downstream of the pipeline we took the output provided by QMASM and rewrote it into a format compatible with QAOA.
